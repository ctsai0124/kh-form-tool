## 一、專案目標

給高雄市學校人事人員用的甄選行政簡化工具：人事人員手動填寫這場甄選的資料 → 產生報到 QR code
→ 考生現場掃碼報到 → 系統自動寄「報到通知＋甄選成績表」給承辦人 → 承辦人只要收信、列印。
**使用者不需要幫考生產報名表**（考生自備），報名表功能已決定整個移除。
**（2026/7/11 更新）原本規劃的「上傳簡章 PDF → 自動判讀」也已決定整個移除**，
判讀準確率不佳、欄位多為手填，人事人員直接手動輸入比較實際。
**（同日再更新）①分頁欄位也精簡了**：拿掉名額、聘任期限、報名時間、報名地點、聯絡人/電話、
報名費、甄試時間、甄試地點、甄試項目與比例、最低錄取分數、成績公告時間、複查截止時間、
報到截止時間這些欄位——這些原本只是餵給「附表一/二/三 報名表/甄選證/切結書」預覽產生器用的，
但那個預覽功能本來就已經決定不做了（考生自備報名表），這些欄位對報到/QR/成績表流程完全沒有
實際作用，所以連同 `renderPreview()`／`previewWrap`／`printBtn`／附表一二三 HTML 樣板一起整個刪除。

**（再更新）學校名稱／職缺類型（radio）／學年度 三個欄位也合併了**，改成單一自由輸入欄位
「簡章名稱」（`f_announcement_title` → `data.announcement_title`），使用者直接打完整的名稱，
例如「高雄市前鎮區前鎮國小115學年度代理教師甄選」。活動代碼改用
`slugForCode(announcement_title) + slugForCode(announcement_no)`。
最終產生的成績表標頭＝`{簡章名稱}人員成績表`（例：「高雄市前鎮區前鎮國小115學年度代理教師甄選
人員成績表」），`.gs` 裡的 `buildScoreSheetPdf()` 已同步改成這個格式，`lookupRecord()` 回傳值
也從 `schoolName` 改名為 `announcementTitle`（提醒：Google 表單/試算表那欄問題標題還是舊的
「學校全銜」，內容已經變成簡章名稱，欄位標題之後可以去表單那邊手動改掉，但不影響程式運作）。
**（7/11 深夜再精簡）「報到 QR code」「即時報到名單」兩個附加工具也從導覽列移除**——
QR 在主頁就地產生＋信裡有附件；名單每次報到都寄累積 xlsx，即時看名單需求消失。
導覽列只剩：`[建立甄選報到][成績表（手動備用）]`。

因為主頁的產生功能依賴 tab2 裡的 `formTemplateInput`（報到表單預填範本）與
`qrAnnouncementNo`／`filterAnnouncementNo`／`csvUrlInput` 等元素；要復原這兩個分頁只要把
導覽按鈕加回去即可。活動代碼已寫進「甄選報到已建立」通知信（含說明），使用者不再需要
從網頁上抄代碼。

**（7/11 晚間排版重整）②③④已降級為「附加工具」**：主流程就是原①分頁一條龍（填資料→
產生QR→自動寄信），②的QR在主頁就地產生、④的正式成績表已自動寄xlsx，③現場看名單是輔助性質。

①分頁現在的欄位：簡章名稱、簡章編號、職缺類型（radio）、通知信箱（＋進階活動代碼）。

## 三、核心設計決策（不要走回頭路）

1. **活動代碼**＝`簡章名稱_簡章編號`（去除符號各取前20字，如
   `高雄市前鎮區前鎮國小115學年度代理教師甄選_第2號第2次`）。**（7/11 再改版）已拿掉隨機碼**：
   原本的隨機碼會導致「頁面載入當下就固定、後填的名稱帶不進去、重新整理就換一組」，
   使用者實測同一場考試測三次產生三個代碼（`__chek`／`__jo0o`／`__ce40`），報到紀錄被拆散、
   成績表無法累積。改成純由簡章名稱＋簡章編號決定（且欄位輸入時即時重算），同一場考試
   不管重新整理或重按產生，代碼永遠相同；不同學校因簡章名稱（含校名+學年度+職缺）不同
   自然不撞號。系統內部比對一律用它。QR code 用 Google 表單「預先填入的連結」把活動代碼
   鎖進網址，考生看不到、改不了。
2. **多校共用同一套表單/試算表/Apps Script**（先 1~2 校試辦），帳號扛在 ctsai0124@gmail.com，
   已知限制：免費 Gmail 每日寄信上限約 100 封。
3. ~~簡章判讀為純本機（pdf.js 抓文字層；掃描件用 Tesseract.js OCR 前 4 頁 + regex）~~
   **已改版（2026/7/11）：整個「上傳簡章 PDF → 自動解析」功能已移除。**
   使用者反映 OCR/regex 判讀準確率太差、欄位多為手填，乾脆拿掉上傳這一步，
   人事人員直接在①分頁的空白欄位手動填寫全部資料。pdf.js／Tesseract.js 的 `<script>` 引入、
   dropzone、parseBtn、OCR 與規則式解析的所有程式碼都已刪除。①分頁一開頁就是空白欄位可直接填，
   不再有「上傳／解析」兩個步驟。
4. `mailto:` 無法夾附件 → 凡是要夾檔案的信一律走 Apps Script 寄。
