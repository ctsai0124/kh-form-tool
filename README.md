# 簡章轉報名表工具

上傳學校甄選簡章 PDF，自動判讀關鍵欄位，並支援從報名到甄選當天的整套流程：

1. **① 簡章解析／報名表** — 判讀簡章 PDF，套印「報名表 / 甄選證 / 切結書」
2. **② 報到 QR code** — 產生連到 Google 表單的 QR code，供考生現場報到掃碼
3. **③ 即時報到名單** — 讀取 Google 試算表（已發布為 CSV），即時顯示報到名單，依科別自動編上准考證號碼
4. **④ 甄選成績表** — 依報到名單自動產生評分表，委員人數可切換 2／3 位，即時計算平均分數，可列印

**完全在瀏覽器端執行，不需要 API key、不需要自架伺服器**
（①②④ 完全不會外流任何資料；③ 會讀取你自己 Google 試算表公開發布的 CSV，本工具只讀不寫）。

## 放到 GitHub Pages 上線（約 5 分鐘）

### 1. 建立一個新的 Repository
1. 登入 GitHub → 右上角 `+` → **New repository**
2. Repository name 隨意取，例如 `kh-form-tool`
3. 選擇 **Public**（GitHub Pages 免費方案需要 Public repo）
4. 建立完成後，把這個資料夾裡的 `index.html` 上傳進去：
   - 網頁上點 **Add file → Upload files**，把 `index.html` 拖進去
   - 或用 git 指令：
     ```bash
     git clone https://github.com/你的帳號/kh-form-tool.git
     cd kh-form-tool
     cp index.html .
     git add index.html
     git commit -m "建立簡章轉報名表工具"
     git push
     ```

### 2. 開啟 GitHub Pages
1. 進入 repo 的 **Settings → Pages**
2. Source 選擇 **Deploy from a branch**
3. Branch 選 `main`，資料夾選 `/ (root)`，按 **Save**
4. 等 1～2 分鐘，頁面上會出現網址，格式類似：
   ```
   https://你的帳號.github.io/kh-form-tool/
   ```

## 一次性設定：建立「萬用報到 Google 表單」

這份表單建一次之後，往後**每一場甄選都重複使用同一份**（甚至多所學校都能共用同一份）。
考生**不需要自己填任何編號**——本工具會把①分頁自動產生的「活動代碼」，用 Google 表單的
「預先填入的連結」功能直接鎖進 QR code 裡，避免多場甄選、多所學校同時報到時混在一起，
也避免考生打錯編號。

1. 到 [forms.google.com](https://forms.google.com) 建立新表單，標題例如「甄選現場報到」
2. 加入以下 4 個「簡答」欄位（建議設為必填）：
   - 活動代碼
   - 姓名
   - 身分證後4碼
   - 報考科別
3. 右上角「回覆」分頁 → 點綠色 Google 試算表圖示 → 建立連結的試算表
4. 打開這份試算表 → **檔案 → 共用 → 發布到網路** → 選擇該工作表分頁、格式選 **CSV** → 發布 → 複製產生的網址
   （這組網址要貼到工具的③分頁「試算表 CSV 網址」欄位）
5. **重點步驟**：回到表單編輯畫面，右上角「⋮」更多選項 → 「取得預先填入的連結」
   → **只在「活動代碼」這一格填入任意文字**（例如 TEMP），其餘欄位留空 → 「取得連結」→ 複製網址
   → 貼到工具②分頁的「表單預填連結範本」欄位（瀏覽器會記住，之後不用再貼）

之後每次要辦新的一場甄選，只需要：
- 在①分頁上傳新的簡章，會自動產生新的「活動代碼」
- 到②分頁，確認活動代碼正確（會自動從①帶入）→ 按「產生 QR code」→ 印出來貼在報到桌
- 到③分頁，把篩選欄位換成新的活動代碼（會自動從①帶入）
- 表單本身、試算表、「預填連結範本」都不用重新設定，只有活動代碼跟著換

**為什麼要「鎖」活動代碼而不是讓考生自己填簡章編號？**
如果讓考生自己在表單上打簡章編號，格式稍有出入（多打一個空格、少打一個字）就會在③分頁篩選不到，
而且沒辦法保證考生一定會抄對；如果多所學校共用同一套系統，不同學校的簡章編號規則也很容易撞號
（兩間學校都可能有「第2號第2次」）。用「預先填入的連結」把①分頁自動產生的「活動代碼」鎖進網址裡，
這格對考生來說是隱形的、改不了的，就算現場同時有好幾場甄選在報到、或好幾間學校共用這套系統，
也不會混在一起。

## 共用後端 + 自動通知：整體設計（給多校共用時）

這套系統目前設計成**所有學校共用同一份 Google 表單／試算表／Apps Script**（用你自己的 Google 帳號架設）。
因為是共用，系統內部不會用「簡章編號」互相比對，而是①分頁解析完後**自動產生一組「活動代碼」**
（學校簡稱＋簡章編號＋4碼隨機碼，例如 `竹滬國小_第2號第2次_a8f3`），所有比對、篩選、通知查找都是用這組代碼，
簡章編號只是給人看的。

**已知的限制（現在只給 1～2 校測試沒問題，之後量大要注意）：**
- 免費 Gmail 帳號每日寄信上限約 100 封；Google Workspace（教育版）帳號約 1500 封
- 所有學校的報到資料（姓名、身分證後4碼）都會進到你這個 Google 帳號的試算表裡
- Apps Script 有每日執行時間總額度限制（consumer 帳號約 90 分鐘/天），正常使用量不會碰到

### 完整資料流

```
①分頁：上傳簡章 → 解析 → 自動產生「活動代碼」→ 填「通知信箱」→ 按「產生」
    → 列印 + 把 {活動代碼, 學校, 通知信箱, 完整資料JSON} 送進「甄選紀錄」表單/試算表

②分頁：貼「表單預填連結範本」→ 活動代碼自動帶入 → 產生 QR code（考生看不到活動代碼，鎖死在網址裡）

【考生報到】掃 QR → 填報到表單(活動代碼已鎖定/姓名/身分證後4碼/科別) → 送出
    → Apps Script 觸發：查「甄選紀錄」試算表拿到通知信箱、算出准考證號碼、
      產生單人成績表 PDF、自動寄信給通知信箱名單

③分頁：貼 CSV 網址 + 活動代碼 → 即時看到報到名單（跟 Apps Script 算的准考證號碼邏輯一致）

④分頁：把③的名單匯入 → 手動輸入委員評分 → 即時算平均 → 列印正式成績表
```

### 設定順序建議（第一次設定，全系統只做一次）

1. 建立「甄選紀錄」表單＋試算表（①分頁「⚙️ 一次性設定」裡的教學）
2. 建立「報到」表單＋試算表（②分頁教學），拿到表單的「預先填入的連結」範本
3. 把 `checkin-notify-apps-script.gs`（這個資料夾裡附的檔案）整份貼進**報到表單**的 Apps Script 編輯器：
   表單編輯畫面 → 右上角「⋮」→ 指令碼編輯器
4. 打開檔案裡的 `CONFIG` 區塊，填入：
   - `RECORD_SPREADSHEET_ID`：步驟 1 那份「甄選紀錄」試算表的 ID（網址 `/d/` 和 `/edit` 中間那串英數字）
   - `RECORD_SHEET_NAME`：該試算表裡實際的回覆分頁名稱（通常是「表單回應 1」，可以打開試算表確認）
5. 左側「觸發條件」（鬧鐘圖示）→ 新增觸發條件：
   - 選取要執行的函式：`onFormSubmitTrigger`
   - 事件來源：來自表單
   - 事件類型：提交表單時
   - 儲存 → 第一次會跳出 Google 帳號授權畫面，選你的帳號 → 允許
6. 用手機掃自己②分頁產生的 QR code，填一筆測試資料送出，檢查通知信箱有沒有收到信、PDF 附件內容對不對

之後每次要辦新的一場甄選，只需要：①上傳簡章、填通知信箱、按「產生」；②確認活動代碼正確、按「產生 QR code」印出來。
表單、試算表、Apps Script 都不用重新設定。

## 各分頁使用注意事項

**① 簡章解析／報名表**
- 3 份掃描版簡章只會 OCR 前 4 頁（大部分學校的關鍵資訊都在前 1～2 頁）
- 這是規則式判讀，非 AI，請務必核對左側欄位再列印
- 第一次使用掃描版簡章時，瀏覽器需要下載 OCR 語言模型（幾 MB），需要網路連線
- 右側「產生（列印＋記錄＋寄信提醒）」按鈕會一次做三件事：
  1. 觸發列印
  2. 把這次解析出的資料（含活動代碼、通知信箱）送進「甄選紀錄」Google 表單（存檔用，跟②③的報到表單是不同份）
  3. 用 `mailto:` 打開一封寫給你自己的信（草稿），提醒內容不會漏記
  - 這三件事都需要先在「⚙️ 一次性設定」裡設定好「紀錄表單預填連結範本」與「提醒 Email」，
    設定方式跟②分頁的「預填連結」概念一樣：建一份有 4 欄位（活動代碼／學校全銜／通知信箱／完整資料）的
    Google 表單，用「取得預先填入的連結」時分別填入 `ANNO_NO`、`SCHOOL`、`NOTIFY_EMAILS`、`FULLDATA`
    這四個標記文字，複製連結貼進去即可，瀏覽器會記住，之後不用再設定
  - 記得在左側欄位填「通知信箱」——這是考生報到後 Apps Script 自動寄信要通知的對象，可以填多人（逗號分隔）
  - 如果只想單純列印不記錄，用旁邊的「單純列印 / 另存 PDF」按鈕即可

**② 報到 QR code**
- 活動代碼是「鎖」進網址裡的，考生完全看不到、改不了這欄，只填姓名/身分證後4碼/科別
- 「表單預填連結範本」只要設定一次，之後每場甄選只需要換活動代碼重新按「產生 QR code」
- 換活動代碼後 QR code 圖案會跟著變，記得列印/更新現場貼的那張

**③ 即時報到名單**
- 「活動代碼」篩選是**完全比對字串**，但因為活動代碼是①分頁自動帶入的，不是人工輸入，不會有打錯字的問題
- Google 發布的 CSV 偶爾會有幾分鐘快取延遲，按「重新整理」按鈕即可重抓最新資料
- 准考證號碼是依「報到時間先後」在**同一科別內**自動從 1 開始編號

**④ 甄選成績表**
- 委員人數可以在最上面切換 2 位／3 位，會即時調整表格欄位
- 平均分數即時計算，「錄取資格」欄位請人事人員自行填寫判斷（工具不自動判定正取/備取）
- 也可以不透過③分頁匯入，直接按「+ 手動新增一列考生」測試格式或處理臨時個案

## 之後想維護／調整判讀規則

①分頁的判讀規則都寫在 `index.html` 裡的 `parseFieldsFromText()` 函式，是一串 regex（樣式比對）。
如果之後發現某個學校的欄位一直抓不到，可以把該欄位附近的簡章文字片段告訴我，我再幫你加規則進去。
