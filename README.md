# 甄選報到系統

給學校人事人員用的甄選行政簡化工具。整個流程只有四步：

1. **填寫甄選資料**：簡章名稱、簡章編號、職缺類型、評審人數、通知信箱
2. **按「產生」**：報到 QR code 就地產生，系統同時自動寄一封信到通知信箱
   （內含這場甄選的資訊、活動代碼、純 QR 圖片、以及可直接列印的 A4 報到海報 PDF）
3. **列印 QR code 海報**，張貼在甄試報到處
4. **考生用手機掃碼報到**（填姓名、身分證後4碼、報考科別）→ 每次報到，系統自動寄
   「成績表」與「評分總表」兩份 Excel 檔到通知信箱，內容是該科別目前**所有已報到考生**
   的累積名單 → 承辦人只要以最新收到的一封為準、列印即可

## 人事人員日常使用

打開網頁 → 照畫面上的流程指引填完四個欄位 → 按「產生」→ 收信，結束。

- 每一場新甄選就是重複上面的動作一次，其他什麼都不用設定
- 「評分總表」只寄一份；評審有幾位就自行加印幾份即可
  （每位考生在總表中會依你選的「評審人數」預留 2 或 3 列評分欄）
- 導覽列右側的「成績表（手動備用）」是備援工具：萬一信件有問題，
  可在該頁手動增加考生列、現場即時填分數並列印

## 為什麼考生不用填簡章編號？——「活動代碼」設計說明

本工具會依你填的資料自動組出一組「活動代碼」（＝簡章名稱＋簡章編號＋職缺類型），
並用 Google 表單的「預先填入的連結」功能把它**鎖進 QR code 的網址裡**。
對考生來說這格是隱形的、改不了的。

好處：
- 考生不需要抄任何編號，也就不會抄錯（多打一個空格、少打一個字都會讓資料對不上）
- 就算現場同時有好幾場甄選在報到、或多所學校共用這一套系統，資料也不會混在一起
  （不同學校的簡章編號很容易撞號，例如兩間學校都有「第2號第2次」；
  但「簡章名稱＋編號＋職缺」的組合不會撞）
- 同一場甄選不管重新整理頁面、重按幾次「產生」，活動代碼都相同，
  所有考生的報到紀錄自然歸在同一場，成績表才能正確累積
- 活動代碼會寫在「甄選報到已建立」通知信裡，日後需要查詢時直接從信裡複製

## 一次性部署設定（只有要「自建一套」的管理者才需要看）

> 目前這個網站已經完成所有設定並內建預設值，**一般使用者完全不需要做以下任何步驟**。
> 以下說明僅供需要用自己的 Google 帳號另外架一套的人參考。

整套系統 = 一頁靜態網頁（本 repo 的 `index.html`）＋ 兩份 Google 表單＋一支 Apps Script：

### 1. 建立「萬用報到表單」（考生掃 QR 後填的那份）

建一次之後，往後每一場甄選、甚至多所學校都重複共用同一份。

1. 到 [forms.google.com](https://forms.google.com/) 建立新表單，標題例如「甄選現場報到」
2. 加入 4 個「簡答」欄位（建議必填）：活動代碼、姓名、身分證後4碼、報考科別
   - 建議把「報考科別」改成**下拉式選單**列出常見科目（國語、英語、數學、自然……＋其他），
     避免考生打「英語」「英文」被系統當成兩個科別、成績表被拆成兩張
3. 「回覆」分頁 → 綠色試算表圖示 → 建立連結的試算表
4. 該試算表 → 檔案 → 共用 → 發布到網路 → 選回覆分頁、格式選 CSV → 發布 → 複製網址
   → 取代 `index.html` 裡的 `DEFAULT_CSV_URL` 常數
5. 回表單編輯畫面 → 右上「⋮」→「取得預先填入的連結」→ **只在「活動代碼」填任意文字
   （例如 TEMP）**，其餘留空 → 取得連結 → 複製網址
   → 取代 `index.html` 裡的 `DEFAULT_FORM_TEMPLATE` 常數

### 2. 建立「甄選紀錄表單」（網頁按「產生」時在背景送出的那份）

1. 再建一份表單，加入 4 個「簡答」欄位：活動代碼、學校全銜、通知信箱、完整資料
   （**必填務必全部關閉**，欄位標題要跟 Apps Script `CONFIG` 裡設定的文字一致）
2. 一樣建立連結的試算表
3. 取得預先填入的連結：四格分別填 `ANNO_NO`、`SCHOOL`、`NOTIFY_EMAILS`、`FULLDATA`
   → 複製網址 → 取代 `index.html` 裡的 `DEFAULT_RECORD_TEMPLATE` 常數

### 3. 部署 Apps Script

1. 打開**報到表單**的指令碼編輯器（擴充功能 → Apps Script），
   貼上 `checkin-notify-apps-script.gs` 全部內容
2. 修改檔案最上方 `CONFIG`：
   - `RECORD_SPREADSHEET_ID`：甄選紀錄**試算表**的 ID
   - `RECORD_SHEET_NAME`：回覆所在分頁名稱（通常「表單回應 1」）
   - `RECORD_FORM_ID`：甄選紀錄**表單本身**的 ID（表單編輯網址 `/forms/d/【這段】/edit`）
3. 手動新增觸發條件：函式 `onFormSubmitTrigger`、來源「來自表單」、事件「提交表單時」
4. 函式下拉選單選 `setupRecordFormTrigger` 執行一次（註冊「甄選紀錄」表單的觸發器）
5. 函式下拉選單選 `authorizeNewScopes` 執行一次（授權雲端硬碟權限，產生 xlsx 附件需要）
6. 過程中跳出的 Google 帳號授權畫面全部允許

## 注意事項

- 寄信配額：免費 Gmail 帳號的 Apps Script 每天最多 100 個收件人
  （建立信 1 封＋每位考生報到 1 封，各乘以通知信箱人數）。超額時信寄不出，
  但報到資料照常寫入試算表，可事後補印。學校教育版 Workspace 帳號配額為 1500/天
- 報到試算表「發布到網路」後，知道 CSV 網址的人可讀取報到考生的姓名與身分證後4碼，
  請自行評估
